<objective>
Close CONS-01 success criteria #1 and #2: produce a re-calibrated graded epi doping profile that lets the 2D solver converge over the full clinical reverse-bias range (down to -50 V) on both 100 µm and 300 µm SVs while reproducing the 1D-twin C-V at the device center within R² >= 0.99 over the FULL clinical bias range (0 to −50 V, sampled at 0, −5, −10, −15, −20, −30, −40, −50 V).

Method (per 26-RESEARCH.md Architecture Pattern 1 + Open Question 2):

1. Port `src/device.py::calibrate_graded_doping` (lines 468-637) to a 2D variant `calibrate_graded_doping_2d` that uses `create_2d_dd_device` for trial builds, `extract_depletion_width_2d_center` for W at the center column, and `reset_devsim_fully` between trials.
2. Cost function blends three known-W targets (0 V, -10 V, -30 V — the validated 1D Petringa data) with a hard convergence penalty at -50 V. Per Open Question 2 the planner accepted "equal weights on known-W targets + 1e3 penalty for divergence at -50 V" — this preserves v3.0 behaviour at low bias (Pitfall P24) while extending the operating envelope.
3. The whole flow is gated on the 26-DIAGNOSIS.md hypothesis from Plan 01: H1 or H3 -> calibrate; H2 -> abort with an error message naming the suspect (mesh / air-buffer / x-symmetry BC) so the user can re-scope before sinking 30-60 min into a doomed fit.
4. Task 3 verifies the full clinical bias range: TestReverseBiasConvergence (6 methods covering -15, -30, -50 V × 2 SVs) AND TestCalibrationCV (R² >= 0.99 at 0, -5, -10, -15, -20, -30, -40, -50 V). The C-V test is wired AFTER the reverse-bias-convergence tests pass, so the 2D solver is known to converge at every voltage before C-V is compared.

Output: `src/device2d.py` has a new `calibrate_graded_doping_2d` function and updated `_N_D_*_DEFAULT` module-level constants reflecting the optimisation result; a runnable `scripts/run_calibration_2d.py` performs the whole pipeline end-to-end; `26-CALIBRATION-RESULT.md` records the chosen values; and two previously-xfail test classes (TestReverseBiasConvergence with 6 methods, TestCalibrationCV with 1 method asserting full-bias R² >= 0.99) now pass with real assertions.

@pytest.mark.slow
def test_2d_vs_1d_cv_centerline(self):
"""CONS-01 SC#2: 2D-center C-V matches 1D C-V over the FULL clinical bias range.

    This test depends on TestReverseBiasConvergence having already passed — the
    2D solver must converge at every voltage in the sweep before C-V comparison
    is meaningful. If a 2D solve fails at -50 V, this test will surface that
    failure as a devsim.error rather than a misleading R² mismatch.
    """
    import numpy as np
    from src.charge_collection_2d import create_2d_dd_device
    from src.drift_diffusion import create_dd_device
    from src.cv_analysis import cv_sweep
    from src.poisson import extract_depletion_width_numerical, extract_depletion_width_2d_center
    from src.devsim_reset import reset_devsim_fully
    from src.device2d import _N_D_JUNCTION_DEFAULT, _N_D_BULK_DEFAULT, _L_TRANSITION_DEFAULT

    # FULL clinical bias range per CONS-01 SC#2 (0 to -50 V)
    voltages = [0.0, -5.0, -10.0, -15.0, -20.0, -30.0, -40.0, -50.0]

    try:
        # 1D twin built with the calibrated 2D defaults so the comparison is
        # apples-to-apples (we are asking whether the 2D solver reproduces a
        # 1D simulation that uses the SAME doping profile).
        reset_devsim_fully()
        dev_1d = create_dd_device(
            device_name=_unique_name() + "_1d",
            doping_profile="graded",
            N_D_junction=_N_D_JUNCTION_DEFAULT,
            N_D_bulk=_N_D_BULK_DEFAULT,
            L_transition=_L_TRANSITION_DEFAULT,
        )
        cv_1d = cv_sweep(dev_1d, V_range=voltages)
        W_1d = np.array(cv_1d["depletion_widths"])

        # 2D center column — manually ramp cathode using cv_sweep fallback pattern
        reset_devsim_fully()
        dev_2d = create_2d_dd_device(
            device_name=_unique_name() + "_2d",
            half_width_um=50.0,
            V_bias=0.0,
            doping_profile="graded",
        )
        import devsim
        import devsim.python_packages.simple_physics as simple_physics
        bias_name = simple_physics.GetContactBiasName("cathode")
        W_2d = []
        current_V = 0.0
        for V_rev in voltages:
            V_cathode_target = -V_rev  # positive cathode = reverse bias
            V = current_V
            # Ramp in 0.5 V steps from current to target
            while V < V_cathode_target - 1e-10:
                V = min(V + 0.5, V_cathode_target)
                devsim.set_parameter(device=dev_2d["device_name"], name=bias_name, value=V)
                try:
                    devsim.solve(type="dc", absolute_error=1e10, relative_error=1e-10, maximum_iterations=40)
                except devsim.error:
                    devsim.solve(type="dc", absolute_error=1e12, relative_error=1e-8, maximum_iterations=100)
            current_V = V_cathode_target
            W_2d.append(extract_depletion_width_2d_center(dev_2d))
        W_2d = np.array(W_2d)

        # R² = 1 - SS_res / SS_tot over the FULL bias range
        SS_res = np.sum((W_2d - W_1d) ** 2)
        SS_tot = np.sum((W_1d - np.mean(W_1d)) ** 2)
        R2 = 1.0 - SS_res / SS_tot if SS_tot > 0 else 0.0

        assert R2 >= 0.99, (
            f"2D-center C-V does not match 1D over full bias range with R² >= 0.99. "
            f"Got R² = {R2:.4f} over voltages {voltages}. "
            f"W_1d (cm) = {W_1d.tolist()}, W_2d (cm) = {W_2d.tolist()}"
        )
    finally:
        reset_devsim_fully()
