  <name>Task 1: Graded doping profile with scipy calibration</name>
  <files>src/device.py</files>
  <action>
Add a `set_graded_doping_profile()` function to device.py that implements an exponential grading in the epi layer:

N_D(x) = N_D_bulk + (N_D_junction - N_D_bulk) \* exp(-(x - x_junction) / L_transition)

for x > x_junction (n-side), where:

- N_D_junction: donor concentration near the metallurgical junction (higher, ~1e15)
- N_D_bulk: donor concentration deep in epi (lower, ~1e13-1e14)
- L_transition: characteristic decay length (cm)

The function should:

1. Accept device_name, region_name, junction_pos, N_A_ionized, N_D_junction, N_D_bulk, L_transition as parameters
2. Use devsim.node_model() with a position-dependent expression for Donors in the n-side
3. Keep Acceptors unchanged (step function in p+ substrate, same as existing)
4. Update NetDoping = Donors - Acceptors

Also add a `calibrate_graded_doping()` function that:

1. Takes target W(V) data points: {0V: 1.7e-4 cm, -10V: 9.5e-4 cm, -30V: 9.73e-4 cm}
2. Uses scipy.optimize.minimize (method='Nelder-Mead') to fit {N_D_junction, N_D_bulk, L_transition}
3. For each trial parameter set: creates a device with graded doping, solves Poisson at equilibrium, ramps voltage to each target bias, extracts numerical W via extract_depletion_width_numerical
4. Minimizes sum of squared relative errors: sum((W_sim - W_exp)^2 / W_exp^2)
5. Returns the optimized parameters dict
6. Uses devsim.delete_device() between trials to avoid state conflicts (see research anti-patterns)

Bounds for optimizer: N_D_junction in [1e14, 1e16], N_D_bulk in [1e12, 1e15], L_transition in [0.5e-4, 5e-4] cm.

Modify create_sic_device() to accept an optional `doping_profile` parameter:

- If `doping_profile="uniform"` (default): use existing set_doping_profile()
- If `doping_profile="graded"`: call set_graded_doping_profile() with provided or default-calibrated parameters
- Add optional kwargs: N_D_junction, N_D_bulk, L_transition

Import scipy.optimize at top of file. Import poisson module functions needed for calibration (setup_poisson, solve_equilibrium, ramp_voltage, extract_depletion_width_numerical).

Create src/drift_diffusion.py with the following functions:

1. `setup_sic_drift_diffusion(device_info)`:
   - Assumes Poisson equilibrium already solved (call setup_poisson + solve_equilibrium first)
   - Creates "Electrons" and "Holes" solution variables via CreateSolution
   - Initializes carriers from equilibrium: get IntrinsicElectrons/IntrinsicHoles node model values from Poisson solution, set as initial Electrons/Holes values
   - Updates Poisson equation to use actual Electrons/Holes instead of Boltzmann approximation: create IntrinsicCharge2 = "kahan3(Holes, -Electrons, NetDoping)", update PotentialEquation node_model to PotentialIntrinsicCharge2
   - Creates SRH recombination: USRH = "(Electrons*Holes - n_i^2)/(taup*(Electrons + n1) + taun\*(Holes + p1))", with derivatives w.r.t. Electrons and Holes
   - Creates ElectronGeneration = "-ElectronCharge _ USRH" and HoleGeneration = "+ElectronCharge _ USRH" with derivatives
   - Creates Bernoulli function via CreateBernoulli(device, region)
   - Creates carrier currents via CreateElectronCurrent(device, region, "mu_n") and CreateHoleCurrent(device, region, "mu_p")
   - Creates electron continuity equation: NCharge = "-ElectronCharge \* Electrons", equation name "ElectronContinuityEquation", variable "Electrons", time_node_model NCharge, edge_model "ElectronCurrent", variable_update "positive", node_model "ElectronGeneration"
   - Creates hole continuity equation: PCharge = "ElectronCharge \* Holes", equation name "HoleContinuityEquation", variable "Holes", same pattern
   - Sets up contact equations for both "anode" and "cathode" using simple_physics.CreateSiliconDriftDiffusionAtContact(device, region, contact, is_circuit=False)
   - Performs initial coupled solve at equilibrium (0V) with tight tolerances: solve(type="dc", absolute_error=1e10, relative_error=1e-10, maximum_iterations=40)

2. `extract_contact_current(device_info, contact="cathode")`:
   - Returns sum of electron and hole contact currents using devsim.get_contact_current()

3. `create_dd_device(doping_profile="graded", **kwargs)`:
   - Convenience function: creates device, sets up Poisson, solves equilibrium, sets up DD
   - Returns device_info augmented with dd_initialized=True
   - Passes doping_profile and kwargs through to create_sic_device()

Follow the exact devsim patterns from the RESEARCH.md code examples. Use the same \_EXP_CLAMP = 700 from poisson.py. All CreateNodeModelDerivative calls must include derivatives for both "Electrons" and "Holes" variables for the Newton solver Jacobian.
