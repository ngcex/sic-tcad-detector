  <action>
    Create `src/device.py`:
    - Function `create_sic_device(device_name="sic_diode", region_name="sic", epi_thickness_cm=10e-4, substrate_thickness_cm=1e-4, N_A=1e19, N_D=None, T=300)`:
      - If N_D is None, use the calibrated value from Plan 01 (store as module constant DEFAULT_N_D).
      - Create 1D mesh using devsim.create_1d_mesh with non-uniform spacing:
        - p+ substrate region: 0 to substrate_thickness_cm, coarse spacing (~100 nm = 1e-5 cm)
        - Junction vicinity: fine spacing (~1 nm = 1e-7 cm) around x = substrate_thickness_cm
        - n- epi region: substrate_thickness_cm to (substrate_thickness_cm + epi_thickness_cm), medium spacing (~50 nm = 5e-6 cm) in bulk, finer near junction
      - Add contacts: "anode" at x=0 (p-side), "cathode" at x=total_length (n-side)
      - Add region spanning entire device
      - Finalize mesh and create device
      - Set 4H-SiC material parameters using SiC4H_Parameters: Permittivity, ElectronCharge, n_i, kT, V_t, mu_n, mu_p. Use doping-dependent mobility from mobility_caughey_thomas.
      - Compute N_A_ionized using ionized_acceptor_concentration(N_A, T) and set doping profile using devsim.node_model: Acceptors on p-side (x < junction_pos), Donors on n-side (x >= junction_pos), NetDoping = Donors - Acceptors.
      - Return a dict with device_name, region_name, junction_pos, epi_thickness_cm, N_D, N_A_ionized, and the SiC4H_Parameters instance.

    Create `src/poisson.py`:
    - Function `setup_poisson(device_info)`:
      - Use `devsim.python_packages.simple_physics` to set up the Poisson equation, initial potential, and contact equations.
      - Follow the devsim diode example pattern: create IntrinsicElectrons/IntrinsicHoles node models, PotentialIntrinsicNodeCharge, set up Poisson equation with NodeModel.
      - OR use simple_physics.CreateSiliconPotentialOnly / similar approach adapted for SiC parameters already set on the device.
      - The key is: use the devsim simple_physics module as much as possible, since SiC parameters are already set on the device. The simple_physics module reads Permittivity, n_i, etc. from the device parameters.
    - Function `solve_equilibrium(device_info)`:
      - Solve at 0V bias using dc solve with appropriate tolerances (absolute_error=1e10, relative_error=1e-10, maximum_iterations=40).
      - If convergence fails, try: (a) increasing max_iter to 100, (b) adjusting damping.
      - Log convergence info.
    - Function `ramp_voltage(device_info, contact_name, V_start, V_end, V_step, **solver_kwargs)`:
      - Ramp bias from V_start to V_end in increments of V_step.
      - At each step, solve and record results.
      - For reverse bias on cathode: V_start=0, V_end=-60, V_step=-0.5 (small steps per RESEARCH.md anti-patterns).
      - Return list of (voltage, solution_data) tuples.
    - Function `extract_electric_field(device_info)`:
      - Get the ElectricField edge model from devsim (or compute from potential gradient).
      - Return (x_positions, E_field_values) arrays.
    - Function `extract_depletion_width(device_info)`:
      - Extract depletion width from the numerical solution. Method: find where |E-field| drops below a threshold (e.g., 1% of peak), or find where net charge density changes sign.
      - Return W in cm.
    - Function `voltage_sweep(device_info, voltages)`:
      - High-level function: solve equilibrium, then ramp through voltages, extracting E-field and W at each voltage.
      - Return structured results dict with voltages, E-fields, depletion widths.

    Create `tests/test_poisson.py`:
    - Test that create_sic_device runs without error and produces a valid devsim device.
    - Test that solve_equilibrium converges (no exception).
    - Test that extract_electric_field returns arrays with correct shape and physically reasonable values (E_max ~ 1e3-1e5 V/cm for moderate bias).
    - Test numerical vs analytical depletion width comparison at 0V: within 20% agreement.
    - Test that depletion width increases with reverse bias.
    - VALIDATION test: compare numerical W at 0V, -10V, -30V against experimental targets (1.7, 9.5, 9.73 um) with 25% tolerance (relaxed for numerical vs experimental).
    - Handle the case where devsim is not installed: skip tests with `pytest.importorskip("devsim")`.
