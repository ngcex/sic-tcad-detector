## Docstring Gap: `poisson.extract_depletion_width`

### Current State

The function at `src/poisson.py:268-297` has this docstring:

```
Extract depletion width using analytical formula.

Uses the one-sided depletion approximation with built-in potential
computed from the device's ionized doping concentrations and n_i.
```

### What's Missing

The docstring does not explicitly warn that this returns an **analytically-derived** W (from the textbook one-sided depletion formula), **not** a W extracted from the numerical Poisson E-field solution. This distinction is critical for Phase 2 drift-diffusion code, which must not assume that `extract_depletion_width` accounts for any numerical effects.

### Module-Level Context

The module docstring (lines 11-16) does explain this design choice:

> "For depletion width computation under bias, uses the analytical one-sided depletion formula (validated against numerical equilibrium at 0V) with punch-through clamping..."

But this is at the module level, not at the function level where a Phase 2 developer would look.

### Recommended Docstring Addition

Add a clear `Note` or `Warning` section to the function docstring:

```python
    Note
    ----
    This returns W computed from the **analytical** one-sided depletion
    approximation (W = sqrt(2*eps*(V_bi-V)/(q*N_D))), NOT a W extracted
    from the numerical Poisson electric field solution. The analytical
    formula was validated against the numerical equilibrium solution at
    0V (agreement within ~20%), but under bias it does not account for
    any numerical effects (e.g., carrier redistribution, non-abrupt
    junction profiles). Phase 2 drift-diffusion code should be aware
    that this is a purely analytical quantity.
```

Also consider adding a `See Also` pointing to `extract_depletion_width_numerical` which does extract W from the numerical solution (but only at equilibrium).

## Common Pitfalls

### Pitfall 1: Notebook JSON Corruption

**What goes wrong:** Editing .ipynb JSON and breaking the notebook format (wrong cell structure, missing metadata).
**Why it happens:** .ipynb files are JSON with specific schema requirements.
**How to avoid:** Load with `json.load`, modify only `cell['source']` list entries, write back with `json.dump(indent=1)`. Do NOT change cell metadata, outputs, or notebook-level metadata.
**Warning signs:** Jupyter refuses to open the notebook after edit.

### Pitfall 2: Matplotlib State Leaks in Tests

**What goes wrong:** Tests create figures that accumulate in memory; test suite slows down or crashes.
**Why it happens:** `plt.subplots()` creates figures that persist until explicitly closed.
**How to avoid:** Use `plt.close('all')` in teardown, or use `@pytest.fixture(autouse=True)` to close figures after each test. Alternatively, always pass an explicit `ax` to avoid implicit figure creation.
**Warning signs:** Memory warnings during test runs.

### Pitfall 3: Matplotlib Backend in CI/Headless

**What goes wrong:** Tests fail with "no display" errors when run without a GUI.
**Why it happens:** Default matplotlib backend requires a display server.
**How to avoid:** The `src/plotting.py` module imports `matplotlib` before `matplotlib.pyplot`, but does not set backend. Tests should use `matplotlib.use('Agg')` before importing pyplot, OR rely on the fact that pytest typically sets Agg automatically. Safest: add `import matplotlib; matplotlib.use('Agg')` at top of test file.
**Warning signs:** ImportError or RuntimeError mentioning Tcl/Tk or display.

### Pitfall 4: Removing "Wrong" Imports

**What goes wrong:** Removing an import that IS actually used, just not obviously (e.g., used via `getattr` or string eval).
**Why it happens:** Automated analysis misses dynamic usage.
**How to avoid:** The analysis above is based on full text search across all notebook cells. Double-check: `compute_ni` and `mobility_caughey_thomas` are not referenced by any string in any cell. `voltage_sweep`, `plot_electric_field_multi`, `plot_doping_profile`, `save_figure` are similarly absent.
**Warning signs:** NameError when running notebook after cleanup.

### Testing a Plotting Function (smoke test pattern)

```python
import matplotlib
matplotlib.use('Agg')
import matplotlib.pyplot as plt
import numpy as np
import pytest

from src.plotting import plot_electric_field

class TestPlotElectricField:
    def teardown_method(self):
        plt.close('all')

    def test_returns_axes(self):
        x = np.linspace(0, 10e-4, 100)
        E = np.linspace(5e4, 0, 100)
        ax = plot_electric_field(x, E, voltage_label="0V")
        assert isinstance(ax, plt.Axes)

    def test_creates_figure_when_no_ax(self):
        x = np.linspace(0, 10e-4, 100)
        E = np.linspace(5e4, 0, 100)
        ax = plot_electric_field(x, E)
        assert len(ax.get_lines()) > 0

    def test_uses_provided_ax(self):
        fig, ax = plt.subplots()
        x = np.linspace(0, 10e-4, 100)
        E = np.linspace(5e4, 0, 100)
        result = plot_electric_field(x, E, ax=ax)
        assert result is ax
```
