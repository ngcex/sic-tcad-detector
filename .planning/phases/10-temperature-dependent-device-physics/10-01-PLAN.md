  <name>Task 1: Add T-dependent material property functions to sic_material.py</name>
  <files>src/sic_material.py</files>
  <action>
Add 5 new public functions below the existing `mobility_caughey_thomas()`. All functions must use parameters from `SiC4H_Parameters` dataclass where applicable. Add new dataclass fields for T-scaling exponents.

1. **Add new fields to SiC4H_Parameters:**
   - `gamma_n: float = -2.40` (electron mobility T exponent, Ayalew)
   - `gamma_p: float = -2.15` (hole mobility T exponent, Ayalew)
   - `alpha_tau: float = 1.72` (SRH lifetime T exponent, Ayalew Z1/2 center)

2. **`bandgap(T, params=None) -> float`:** Varshni equation using params.E_g_0, E_g_alpha, E_g_beta. If params is None, use SiC4H_Parameters(). Must return exactly params.E_g (3.26) at T=300.

3. **`intrinsic_concentration(T, params=None) -> tuple[float, float, float, float]`:** Calibrated n_i with T-dependence. Use the calibration factor approach: `n_i(T) = params.n_i_300 * compute_ni(T) / compute_ni(300)`. Also return NC(T), NV(T), E_g(T). At T=300, must return exactly (params.n_i_300, NC_300_computed, NV_300_computed, params.E_g). The NC/NV values come from compute_ni, so they won't exactly match NC_300/NV_300 in the dataclass (that's fine, the dataclass values are reference only).

4. **`mobility_caughey_thomas_T(N_total, T, carrier="electron", params=None) -> float`:** Extends existing Caughey-Thomas with T-scaling on mu_max only: `mu_max(T) = mu_max(300) * (T/300)^gamma`. mu_min stays constant. At T=300 must return identical values to existing `mobility_caughey_thomas()`.

5. **`effective_dos(T, params=None) -> tuple[float, float]`:** Returns (NC(T), NV(T)) using T^(3/2) scaling from compute_ni(). Just wraps compute_ni to extract NC, NV.

6. **`srh_lifetime(T, carrier="electron", params=None) -> float`:** Power-law T scaling: `tau(T) = tau_300 * (T/300)^alpha_tau`. At T=300 returns exactly params.tau_n or params.tau_p.

Keep the existing `compute_ni()` and `mobility_caughey_thomas()` functions unchanged for backward compatibility. The new functions are additions, not replacements.
