  <action>
Create `src/temperature_sweep.py` with these functions:

1. **`sweep_iv_vs_temperature(temperatures, V_reverse=-30, **device_kwargs) -> pd.DataFrame`:\*\*
   - For each T in temperatures: create DD device at T, ramp to V_reverse, extract reverse current
   - Return DataFrame with columns: T, I_reverse, n_i, mu_n, mu_p, E_g
   - Use `create_dd_device()` and voltage ramping from `drift_diffusion.py`
   - Each device needs a unique name (use uuid). Clean up devsim devices after each T to avoid memory issues.

2. **`sweep_cv_vs_temperature(temperatures, voltages=None, **device_kwargs) -> dict`:\*\*
   - For each T: create device, extract C-V curve, get W at each voltage
   - Return dict with 'temperatures' array and 'W' 2D array (T x V), plus 'voltages'
   - Default voltages: [0, -5, -10, -20, -30]

3. **`sweep_cce_vs_temperature(temperatures, voltages=None, method="hecht", **device_kwargs) -> pd.DataFrame`:\*\*
   - For each T: compute CCE at given voltages using Hecht or DD method
   - Return DataFrame with T, V, CCE columns (long format for easy plotting)
   - For Hecht method: use hecht_cce with T parameter and appropriate depletion width from CV
   - For DD method: use compute_cce_vs_bias if available

4. **`extract_temperature_coefficient(temperatures, values, quantity_name="CCE") -> dict`:**
   - Linear regression (scipy.stats.linregress) of values vs T
   - Return dict: slope, intercept, r_squared, unit (e.g., "%/K" for CCE, "pA/K" for current)
   - slope = dQuantity/dT = temperature coefficient

- Keep sweep tests lightweight (3 temperatures max, fast convergence settings) to avoid long test times

Use `import pandas as pd` -- check if pandas is already a dependency. If not, the notebook already uses it (matplotlib/pandas are standard for Jupyter). Add to imports.
