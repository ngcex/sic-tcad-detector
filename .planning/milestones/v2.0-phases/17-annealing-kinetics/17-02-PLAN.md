```
    fluence,

    epi_thickness_cm=10e-4,
    alpha_range_cm=15e-4,
    generation_rate=1e18,

    lifetime_model="linear",
    damage_params=None,
    anneal_params=None,
) -> dict:
```

Implementation:

1. Import compute_annealed_params, AnnealingParams from src.radiation_damage
2. Follow the same "fresh device per point" pattern as cce_vs_fluence but for a SINGLE fluence+anneal point
3. Create device using create_sic_device with staged creation pattern (same as cce_vs_fluence internals)
4. Call compute_annealed_params to get post-anneal device params
5. Apply to device via apply_damaged_params
6. Solve and compute CCE (same pattern as inner loop of cce_vs_fluence)
7. Return dict with: cce, fluence, T_anneal, t_anneal, V_bias, energy_MeV, f_Z12, f_EH67, f_EH4, tau_n, tau_p

Also add cce_anneal_vs_temperature:

```
def cce_anneal_vs_temperature(

    T_anneal_range,
    t_anneal,
    V_bias=-40.0,
    ...same device params as cce_post_anneal...
) -> dict:
```

Sweeps over annealing temperatures at fixed fluence and anneal time, returning arrays of T_anneal and cce_values. Uses fresh device per temperature point. This enables plotting CCE recovery vs annealing temperature.
