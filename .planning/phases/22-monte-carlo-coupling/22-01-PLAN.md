```python
def load_cce_let_table(filepath):
    """Load CCE(LET) table and provide interpolation function.

    Returns
    -------
    cce_interp : callable
        Function cce_interp(LET_keV_um) -> CCE, using log-linear
        interpolation on log10(LET).
    metadata : dict
        Full JSON data including geometry, bias, and raw arrays.
        Keys: geometry (dict with half_width_um, epi_um), bias_V, x_ion_um,
              LET_keV_um (list), CCE (list), Q_generated_fC (list),
              Q_collected_fC (list), t_collection_ns (list)
    """

def ion_track_generation_2d(device_info, LET_keV_um, x_ion_cm=0.0,
                            track_sigma_cm=1e-4, E_pair_eV=8.4):
    """Create 2D generation profile for a single ion track.

    Converts LET to a spatially-resolved charge generation profile on the
    2D mesh. The ion track is vertical (along y) through the epi layer,
    with a Gaussian lateral profile centered at x_ion_cm.

    Parameters
    ----------
    device_info : dict
        2D device info from create_2d_dd_device.
    LET_keV_um : float
        Linear energy transfer (keV/um).
    x_ion_cm : float
        Lateral position of ion track (cm). Default: 0.0 (center).
    track_sigma_cm : float
        Gaussian radial width of track (cm). Default: 1e-4 (1 um).
    E_pair_eV : float
        Electron-hole pair creation energy (eV). Default: 8.4 (4H-SiC).

    Returns
    -------
    generation : ndarray
        Generation density at each mesh node (pairs/cm^3).
    Q_generated_C_per_cm : float
        Total generated charge per unit z-depth (C/cm).
    """
```

Create src/mc_coupling.py with the following public functions, following the architecture patterns from 22-RESEARCH.md:

1. **Unit conversion constants and function:**
   - UNIT_FACTORS dict: mm_to_cm=0.1, cm_to_cm=1.0, um_to_cm=1e-4, MeV_to_keV=1e3, keV_to_keV=1.0, eV_to_keV=1e-3
   - `convert_units(df, pos_unit="mm", energy_unit="MeV")` -> modifies x_cm/y_cm/z_cm and edep_keV columns in-place, returns df
   - Raise ValueError for unknown unit strings

2. **load_mc_events_csv(filepath, column_map=None, pos_unit="cm", energy_unit="keV"):**
   - Read CSV via pandas.read_csv
   - Default column_map: {"event_id": "event_id", "x_cm": "x", "y_cm": "y", "z_cm": "z", "edep_keV": "edep"}
   - Rename columns per column_map (map from standard name to CSV column name)
   - Apply unit conversion if pos_unit != "cm" or energy_unit != "keV"
   - Return DataFrame with standardized columns: event_id, x_cm, y_cm, z_cm, edep_keV
   - Log number of events and total rows loaded

3. **list_root_trees(filepath):**
   - Return list of TTree names via uproot.open(filepath).keys()
   - Filter to only TTree objects (exclude non-TTree items)
   - Provide helpful error message if file not found

4. **load_mc_events_root(filepath, tree_name="Hits", branch_map=None, pos_unit="mm", energy_unit="MeV"):**
   - Default branch_map: {"event_id": "EventID", "x_cm": "PosX", "y_cm": "PosY", "z_cm": "PosZ", "edep_keV": "Edep"}
   - branch_map maps standard name -> ROOT branch name
   - Read via uproot.open(filepath)[tree_name].arrays(branches, library="np")
   - Build DataFrame, apply unit conversion (Geant4 default: mm, MeV)
   - Return standardized DataFrame same as CSV loader
   - Support optional branches: if "particle" branch exists in map and in tree, include it

5. **events_to_charge_profiles(events_df, device_info, sv_thickness_um=10.0, x_ion_cm=0.0, track_sigma_cm=1e-4, E_pair_eV=8.4):**
   - Convert MC energy deposition events to 2D mesh charge generation profiles (MCCP-03)
   - Group events_df by event_id, sum edep_keV per event
   - Compute effective LET per event: LET_keV_um = total_edep_keV / sv_thickness_um
   - For each event, call `ion_track_generation_2d(device_info, LET_keV_um, x_ion_cm, track_sigma_cm, E_pair_eV)` from single_particle.py to produce a 2D mesh charge generation profile
   - Import ion_track_generation_2d at top of function (from src.single_particle import ion_track_generation_2d)
   - Return list of dicts, one per event: {"event_id": int, "LET_keV_um": float, "generation": ndarray, "Q_generated_C_per_cm": float}
   - Log progress every 100 events (this is slower than the fast-path CCE lookup since it calls the mesh generation for each event)
   - Note in docstring: this is the "full path" for mesh-resolved profiles; for fast batch processing of 1000+ events, use process_mc_ensemble() instead

6. **process_mc_ensemble(events_df, cce_interp, sv_thickness_um=10.0):**
   - Group events_df by event_id, sum edep_keV per event
   - Compute effective LET = total_edep_keV / sv_thickness_um (keV/um)
   - Filter out events with edep <= 0 (log count of zero-energy events)
   - Warn if any LET falls outside typical range [0.01, 10000] keV/um
   - Apply cce_interp to each event's LET (vectorized via np.array comprehension)
   - Compute collected energy: E_collected_keV = CCE \* total_edep_keV
   - Return dict with keys: event_energies_keV (array), event_LET_keV_um (array), event_CCE (array), event_collected_keV (array), n_events (int), n_zero_energy (int)

7. **pulse_height_distribution(collected_energies_keV, n_bins=200, e_min=None, e_max=None):**
   - Filter out events with collected energy <= 0
   - Auto-detect e*min (0.5 * min) and e*max (1.5 * max) if not provided
   - Log-spaced bins via np.logspace
   - Compute histogram via np.histogram
   - Bin centers as geometric mean of edges
   - Return dict with keys: bin_centers_keV, counts, bin_edges_keV

Module-level: use logging.getLogger(**name**). Docstrings for all public functions following existing project style (NumPy docstring format). Import uproot only inside the ROOT functions (lazy import) so the module works without uproot installed for CSV-only use.

IMPORTANT: The column_map convention must be consistent: keys are standard column names, values are source column names in the CSV/ROOT file. This is the reverse of what the research doc shows -- pick one convention and be consistent across both loaders.
