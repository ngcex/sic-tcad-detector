### Forward Integration Risk (Phase 19 → Phase 20)

**`poisson.py::extract_electric_field` (line 265)** uses `x_nodes[:-1]` / `x_nodes[1:]` midpoint computation — valid for 1D sorted arrays only. If Phase 20 inherits the 1D `voltage_sweep` code path and calls this on a 2D device, it will silently return malformed results. The 2D equivalent is `extract_center_slice` + `np.gradient` from `plotting2d.py`.

Similarly, `extract_depletion_width` and `extract_depletion_width_numerical` in `poisson.py` assume 1D topology.

**Recommendation:** Phase 20 planning must add a `device_info["dimension"]` guard or use the 2D-aware functions from `plotting2d.py`.
