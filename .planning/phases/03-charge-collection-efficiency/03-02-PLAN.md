  <files>src/charge_collection.py, src/drift_diffusion.py</files>
  <action>
Add the following functions to src/charge_collection.py (extending the module from Plan 01):

1. `add_generation_to_dd(device_info, generation_values)`:
   Add spatially-varying radiation generation rate to the existing DD equations.
   - device_info: dict from create_dd_device (must have dd_initialized=True).
   - generation_values: array of G(x) at each node (cm^-3 s^-1).
   - First, create a "RadGenRate" node parameter using devsim.node_solution and set_node_values (NOT CreateNodeModel, since this is data not an expression).
     Actually, the cleanest approach: use devsim.set_node_values to set "RadGenRate" after creating it as a node model initialized to "0".
     Step 1: CreateNodeModel(device, region, "RadGenRate", "0") to register the model name.
     Step 2: devsim.set_node_values(device=device, region=region, name="RadGenRate", values=list(generation_values)) to set actual values.
   - Update ElectronGeneration: Gn = "-ElectronCharge _ USRH + ElectronCharge _ RadGenRate"
     This adds +q\*G (creates electrons). Sign follows existing convention where ElectronGeneration positive = electron creation.
   - Update HoleGeneration: Gp = "+ElectronCharge _ USRH - ElectronCharge _ RadGenRate"
     This adds -(-q)\*G = creates holes. Sign: HoleGeneration for holes uses opposite sign convention.
   - Recreate derivatives for Electrons and Holes (RadGenRate has no carrier dependence, so derivatives of RadGenRate terms are zero, but USRH derivatives must be preserved).
   - Re-call devsim.equation() for both continuity equations to update node_model references.
     CRITICAL: Test that adding generation increases carrier density (not decreases). The research warns about sign convention errors.

2. `compute_cce_from_dd(device_info, generation_values, contact="cathode")`:
   Extract CCE from a solved DD device with generation.
   - I_collected = abs(extract_contact_current(device_info, contact))
   - I_generated = Q \* np.trapz(generation_values, x_nodes) where x_nodes from devsim.get_node_model_values "x"
   - CCE = I_collected / I_generated, clipped to [0, 1].
   - Q = 1.602e-19 C.

3. `cce_vs_bias(V_range, epi_thickness_cm=10e-4, alpha_range_cm=15e-4, generation_rate=1e18, device_kwargs=None)`:
   Compute CCE vs reverse bias voltage for alpha particle irradiation.
   - Create a fresh DD device with calibrated graded doping (N_D_junction=2.90e15, N_D_bulk=8.50e13, L_transition=1.0e-4) per Phase 2 calibration.
   - Generate alpha particle profile using alpha_generation_profile, scaled to generation_rate (cm^-3 s^-1). This is low injection since generation_rate << N_D ~ 1e13-1e15.
   - For each voltage in V_range (negative = reverse bias):
     a. Ramp bias to target voltage (incremental steps like iv_sweep does)
     b. Add generation rate (or update if already added)
     c. Solve DD steady-state
     d. Extract CCE
   - Return dict with keys: voltages, cce_values, I_collected, I_generated.
   - Use unique device_name with uuid to avoid devsim name conflicts.
   - Delete device after sweep (devsim.delete_device).
   - IMPORTANT: Ramp voltage FIRST without generation to establish bias point, THEN add generation and re-solve. This avoids convergence issues from simultaneous bias+generation changes.

Also add to src/drift_diffusion.py a helper `ramp_bias(device_info, V_target, contact="anode", V_step=0.5)` if not already present -- a function that incrementally ramps from current bias to target, similar to what iv_sweep does internally but without collecting I-V data. Check if iv_sweep can be reused; if not, extract the ramping logic.
</action>
