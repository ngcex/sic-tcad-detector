  <action>
**In src/device.py**, add `apply_damaged_params(device_info, damaged_params)` function:
- Takes device_info dict (from create_sic_device, BEFORE Poisson setup) and damaged_params dict (from compute_damaged_params)
- Overrides SRH lifetimes: `devsim.set_parameter(device, region, name="taun", value=damaged_params["tau_n"])` and same for "taup"
- Overrides doping: get x_nodes from device, build full Donors array (zeros in p+ substrate where x < junction_pos, damaged_params["N_D_profile"] in epi where x >= junction_pos), set via `devsim.set_node_values` for "Donors"
- Recompute NetDoping: get Acceptors, compute net = donors - acceptors, set via `set_node_values` for "NetDoping"
- IMPORTANT: This function must be called AFTER `create_sic_device()` but BEFORE `setup_poisson()` and `solve_equilibrium()`. The research identifies this as the cleanest approach (Pattern 2 pitfall: equilibrium must be solved with the damaged doping, not the pristine doping). Document this in the docstring.

**In src/charge_collection.py**, add two functions:

1. `cce_vs_fluence(fluence_range, V_bias=-40.0, epi_thickness_cm=10e-4, alpha_range_cm=15e-4, generation_rate=1e18, energy_MeV=62.0, lifetime_model="linear", damage_params=None)`:
   - Follow the "fluence-as-temperature" pattern from temperature_sweep.py: fresh device per fluence point, UUID device names, try/finally with delete_device
   - STAGED device creation (NOT create_dd_device): call create_sic_device -> apply_damaged_params -> setup_poisson -> solve_equilibrium -> setup_sic_drift_diffusion. This is critical per Research Open Question 1.
   - For pristine baseline: extract pristine_tau_n/tau_p from srh_lifetime(300.0, "electron"/"hole"), extract pristine_N_D_profile from a reference device (create, extract epi-only Donors where x >= junction_pos, delete)
   - For each fluence: compute_damaged_params -> staged device creation with apply_damaged_params -> ramp_bias to V_bias on anode -> add_generation_to_dd -> devsim.solve -> compute_cce_from_dd -> record CCE -> delete_device
   - Use logging.getLogger(**name**) for progress: log fluence and CCE at each point
   - Return dict: {"fluences": np.array, "cce_values": np.array, "V_bias": float, "energy_MeV": float, "lifetime_model": str}
   - Use np.geomspace for internal fluence grid recommendations in docstring (caller provides fluence_range)

2. `cce_vs_bias_at_fluence(V_range, fluence, epi_thickness_cm=10e-4, alpha_range_cm=15e-4, generation_rate=1e18, energy_MeV=62.0, lifetime_model="linear", damage_params=None)`:
   - Creates ONE damaged device at the given fluence using staged creation pattern
   - Sweeps bias voltage on that single device (reuse the bias sweep pattern from cce_vs_bias: sort voltages, ramp incrementally)
   - Return dict: {"voltages": np.array, "cce_values": np.array, "fluence": float, ...}
   - try/finally cleanup with delete_device

Key anti-patterns to avoid (from Research):

- Do NOT reuse devices across fluence points in cce_vs_fluence
- Do NOT use create_dd_device() directly -- use staged creation so damaged doping is applied before Poisson equilibrium
- Do NOT modify radiation_damage.py to import devsim
- Do NOT forget to apply BOTH lifetime AND doping damage together
