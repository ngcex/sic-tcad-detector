  <action>
Create src/charge_collection_2d.py with these functions:

1. **integrate_over_mesh_2d(device_info, node_values) -> float**
   - Vectorized triangle-area integration over the 2D triangular mesh.
   - Get x, y from devsim node model values. Get element connectivity via `devsim.get_element_node_list(device=device, region=region)`.
   - Reshape elements into (N_tri, 3) array. Compute triangle areas via cross product: `0.5 * abs((x1-x0)*(y2-y0) - (x2-x0)*(y1-y0))`. Average node values per triangle. Return sum(v_avg \* areas).
   - This returns the integral in cm^2 units (since x,y are in cm).

2. **create_2d_dd_device(half_width_um=50.0, V_bias=50.0, device_name=None) -> dict**
   - Convenience wrapper: calls create_sic_2d_device (with uuid-based name if device_name is None), setup_poisson, solve_equilibrium, setup_sic_drift_diffusion, sets dd_initialized=True, ramps bias via ramp_bias(device_info, V_target=V_bias, contact="cathode", V_step=0.5).
   - Returns device_info dict.
   - Use uuid4().hex[:8] for device name uniqueness (same pattern as cce_vs_bias in charge_collection.py).

3. **compute_cce_2d(device_info, generation_values, contact="cathode") -> float**
   - 2D version of compute_cce_from_dd. Uses integrate_over_mesh_2d for I_generated = Q \* integral(G dA). Uses extract_contact_current for I_collected. Returns CCE clipped to [0, 1].
   - Q = 1.602e-19.

4. **cce_lateral_scan(device_info, n_points=20, gen_rate=1e18, stripe_sigma_cm=2e-4, contact="cathode") -> dict**
   - Compute CCE at n_points lateral positions from x=0 (center) to x=half_width_cm (edge).
   - At each x_pos: create generation G(x,y) = gen_rate _ exp(-y/alpha) _ exp(-0.5\*((x - x_pos)/stripe_sigma_cm)^2) where y is measured from junction (subtract substrate_thickness_cm), alpha = 15e-4 cm (alpha particle range, same as 1D default). Generation is zero for y < junction_pos (substrate region).
   - Call add_generation_to_dd, solve DC, compute CCE via compute_cce_2d.
   - After each position, reset generation to zero and re-solve to restore baseline.
   - Return dict with keys: "x_positions_cm", "x_positions_um", "cce_values", "edge_to_center_ratio" (cce at last point / cce at first point).

5. **cce_heatmap_2d(device_info, cce_lateral, cce_1d_depth=None) -> dict**
   - Construct 2D CCE map using factored form: CCE(x,y) = f_edge(x) \* CCE_1d(y).
   - f_edge(x) is interpolated from the lateral scan (cce_lateral["cce_values"] normalized by center value).
   - If cce_1d_depth is None, assume uniform depth collection (f_depth = 1 in depletion, 0 outside). Otherwise use provided depth profile.
   - Return dict: "x_nodes_cm", "y_nodes_cm", "cce_map" (per-node CCE values), "active_fraction" (fraction of epi area with CCE > 0.5).

6. **compare_cce_2d_vs_1d(half_width_um=50.0, V_bias=50.0, gen_rate=1e18) -> dict**
   - Creates a 2D device and a 1D device, applies uniform epi generation to both, computes CCE.
   - For 1D: use create_dd_device from drift_diffusion.py with same parameters, ramp_bias to same V_bias on anode (negative), use compute_cce_from_dd.
   - For 2D: use create_2d_dd_device, apply uniform generation across entire epi (G=gen_rate for y > junction_pos, 0 otherwise), compute via compute_cce_2d.
   - Also runs cce_lateral_scan on the 2D device to get edge effect data.
   - Compute active_to_geometric_ratio = (integral of CCE(x) dx from 0 to half_width) / (half_width \* CCE_center), using trapezoidal rule on the lateral scan.
   - Return dict: "cce_1d", "cce_2d_center", "cce_2d_full_area", "active_to_geometric_ratio", "lateral_scan" (the cce_lateral_scan result), "half_width_um", "V_bias".
   - Delete both devices in a finally block (devsim.delete_device).

Important implementation notes:

- Import from src.device2d, src.poisson, src.drift_diffusion, src.charge_collection (add_generation_to_dd only).
- Do NOT modify any existing modules. charge_collection.py remains frozen for 1D.
- Use logging (logger = logging.getLogger(**name**)) for progress messages during lateral scans.
- For ramp_bias in 2D, use contact="cathode" (reverse bias on cathode for p+/n- diode in 2D convention -- same as Phase 19 Poisson validation pattern).

- The symmetry factor of 2 cancels in CCE ratios (both I_collected and I_generated are for half-device). Document this in docstrings.
