```python
def get_triangulation(device, region):
    x = np.array(devsim.get_node_model_values(device=device, region=region, name="x"))
    y = np.array(devsim.get_node_model_values(device=device, region=region, name="y"))
    elements = devsim.get_element_node_list(device=device, region=region)
    triangles = np.array(elements)
    return mtri.Triangulation(x, y, triangles)
```

From 19-RESEARCH.md - center slice extraction:

```python
def extract_center_slice(device, region, x_center=0.0, tol=1e-8):
    x = np.array(devsim.get_node_model_values(..., name="x"))
    y = np.array(devsim.get_node_model_values(..., name="y"))
    pot = np.array(devsim.get_node_model_values(..., name="Potential"))
    mask = np.abs(x - x_center) < tol
    return y[mask], pot[mask]
```

3. **`plot_potential_2d(device, region, ax=None, levels=50, cmap="RdBu_r")`** -> (fig, ax)
   - Use get_triangulation + tricontourf
   - Colorbar with label "Potential (V)"

   - Invert y-axis (depth increases downward)
   - Set aspect ratio appropriately (NOT equal -- device is much wider than deep)

4. **`plot_efield_2d(device, region, ax=None, levels=50, cmap="viridis")`** -> (fig, ax)
   - Compute E-field magnitude from potential gradient:
     - Create node model "Ey" = "-((Potential@n0 - Potential@n1) / NodeVolume1)" is NOT straightforward in 2D
     - Instead, compute E_y numerically: use scipy.interpolate.griddata to interpolate potential onto a regular grid, then np.gradient to get E_y and E_x components
     - OR use devsim's `element_from_edge_model` to convert the edge E-field model into an element model, then extract
     - Simplest robust approach: after Poisson solve, the edge model "ElectricField" exists. Use `devsim.element_from_edge_model(edge_model="ElectricField", ...)` to get element-averaged E-field. Then for visualization, compute E_y from finite differences on the Potential node model using np.gradient on a regular grid interpolated from the triangular mesh.
   - Actually, the SIMPLEST approach for MESH-03 visualization: compute |E| = sqrt(Ex^2 + Ey^2) from the potential gradient using node-based finite differences via scipy.interpolate.LinearNDInterpolator + np.gradient on a regular grid, then plot with pcolormesh. But for tricontourf we need node values.
   - **Recommended approach**: After Poisson solve, compute E_y at each node using the devsim formula: create an edge model "Efield" = "(Potential@n0 - Potential@n1) \* EdgeInverseLength", then use `devsim.element_from_edge_model` + averaging back to nodes. If this is too complex, use the simpler approach of extracting potential on a regular grid and computing gradient numerically.
   - For the 1D-vs-2D validation (MESH-02), the E-field comparison uses the center-slice potential: extract potential along center column, compute E_y = -dPotential/dy via np.gradient.
   - Colorbar, labels, inverted y-axis same as potential plot.

5. **`validate_2d_vs_1d(device_info_2d, device_info_1d)`** -> dict
   - Extract center-column potential from 2D using extract_center_slice
   - Extract potential from 1D using get_node_model_values with "x" and "Potential"
   - Interpolate 1D potential onto 2D y-coordinates using np.interp (1D depth = x, 2D depth = y)
   - Compute max relative error: max(|V_2d - V_1d|) / max(|V_1d|)
   - Compute E-field from both: E = -dV/dy via np.gradient, then max relative E-field error
   - Return {"potential_max_rel_error": float, "efield_max_rel_error": float, "pass": bool}
   - "pass" = True if both errors < 0.01 (1%)

6. **`plot_doping_2d(device, region, ax=None)`** -> (fig, ax)
   - Plot log10(|NetDoping|) as tricontourf
   - Sign indicated by separate contour or annotation
   - Useful for verifying MESH-04 visually
