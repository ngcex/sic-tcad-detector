**1. ion_track_generation_2d(device_info, LET_keV_um, x_ion_cm=0.0, track_sigma_cm=1e-4, E_pair_eV=8.4)**

- Convert LET (keV/um) to linear pair density: pairs_per_cm = LET_keV_um _ 1e3 / E_pair_eV _ 1e4
- Get x, y node coordinates from devsim
- Gaussian lateral profile centered at x_ion_cm with sigma=track_sigma_cm, normalized to 1/cm
- Track exists only in epi region: y >= substrate_thickness_cm AND y <= substrate_thickness_cm + epi_thickness_cm
- Return (generation_profile_array, Q_generated_C_per_cm) where Q_generated is computed via integrate_over_mesh_2d(device_info, generation) \* 1.602e-19
- IMPORTANT: generation has units pairs/cm^3. Q_generated = integrate_over_mesh_2d(generation) \* q_electron gives C/cm for 2D.

**2. simulate_single_particle(device_info, generation_profile, dt_inject=1e-12, t_collect=200e-9, dt_min=1e-13, dt_max=1e-9, contact="cathode")**

- Use Approach A from research: generation-pulse method
- Step 1: Initialize transient state -- set zero generation via add_generation_to_dd, then devsim.solve(type="transient_dc", absolute_error=1e10, relative_error=1e-10, maximum_iterations=30)
- Step 2: Record dark current I_dark = extract_contact_current(device_info, contact)
- Step 3: Inject charge as short pulse -- G_rate = generation_profile / dt_inject, add_generation_to_dd(device_info, G_rate), devsim.solve(type="transient_bdf1", tdelta=dt_inject, absolute_error=1e10, relative_error=1e-8, maximum_iterations=100)
- Step 4: Zero generation -- add_generation_to_dd(device_info, zeros), then BDF1 time-step loop
- Adaptive dt: dt = max(dt_min, min(dt_max, t \* 0.1)) -- the 10% rule from research
- Early termination: |I(t) - I_dark| < 0.01 * |I_peak - I_dark| AND t > 10*dt_inject
- Record times[] and currents[] arrays
- Return dict: {"times": np.array, "currents": np.array, "Q_collected": float, "I_dark": float, "I_peak": float}
- Q_collected = np.trapezoid(np.abs(currents) - np.abs(I_dark), times) -- use absolute values for sign safety
- For convergence failures during BDF1 steps: retry with relaxed tolerances (absolute_error=1e12, relative_error=1e-6, maximum_iterations=200). If still fails, log warning and terminate early.

**3. analyze_current_pulse(times, currents, I_dark)**

- Extract: Q_collected (C/cm), I_peak, t_peak, t_collection (time to 95% charge)
- As specified in research code example

**4. build_cce_let_table(half_width_um=50.0, V_bias=50.0, n_let_points=40, let_min=0.1, let_max=1000.0, x_ion_cm=0.0)**

- LET_values = np.logspace(log10(let_min), log10(let_max), n_let_points)
- For each LET: create fresh device via create_2d_dd_device, generate track, simulate, extract CCE
- CRITICAL: delete device after each simulation (devsim.delete_device) to avoid global solver coupling
- CCE = Q_collected / Q_generated
- Return pandas DataFrame with columns: LET_keV_um, Q_generated_fC, Q_collected_fC, CCE, t_collection_ns
- Wrap each simulation in try/except: on failure, log warning, record CCE=NaN, continue to next LET
- Log progress every 5 LET values

**5. save_cce_let_table(table_df, filepath, geometry_info=None)**

- Save as JSON with metadata: geometry (half_width_um, epi_um), bias_V, x_ion_um, columns as lists
- Use json.dump with indent=2

**6. load_cce_let_table(filepath)**

- Load JSON, return (interpolation_function, metadata_dict)
- Interpolation: log-linear via np.interp on log10(LET)

Module-level imports: numpy, pandas, json, logging, devsim, and from src modules: create_2d_dd_device, integrate_over_mesh_2d from charge_collection_2d; add_generation_to_dd from charge_collection; extract_contact_current from drift_diffusion.

Use logging (logger = logging.getLogger(**name**)) throughout for progress and warnings. Follow existing codebase style (docstrings, type hints via docstring not annotations).
